Guitar Top Bracing Tools

Modal analysis · inverse bracing design

Two companion tools for guitar (and violin) top bracing work, both running entirely in the browser — nothing is uploaded anywhere. Bracing Designer starts from measured data and searches for bracing that gets a top to do something specific; Bracing Visualizer starts from a bracing pattern you already have and shows you what it actually does.

Inverse design

Bracing Designer

Imports a roving-hammer modal dataset (ObieWebApp format), lets you set target frequencies against the measured baseline, then searches several bracing topologies — X-brace, fan, falcate, and free-topology variants — for the one that gets closest to your targets while respecting a bridge deflection limit.

  • Import Data → Plate & Load → Extract Modes → Generate Bracing → Results
  • Simulated-annealing search, running locally via Pyodide/scipy
  • Exports a project file the Visualizer can open directly

Forward analysis

Bracing Visualizer

Trace or build a soundboard, add braces by hand, and watch the modal response update as you edit — mode shapes, frequencies, and an animated 3D view of how the top actually moves.

  • Soundboard → Braces → Modal Analysis → 3D View
  • Direct manual control over every brace's position and dimensions
  • Opens project files exported from Bracing Designer
Open Bracing Visualizer →
These two are meant to be used together. A natural loop: bring measured data into Designer, solve for bracing that hits your targets, then use "Export as bracing-tool project…" on any result and open it straight in Visualizer — the file formats already match — to inspect, hand-tune, or animate the winning layout.
Runs entirely client-side — no data leaves your browser. See each tool's own Documentation for details on its models and assumptions.
